# 採点・面接の履歴をスプレッドシートに残す設定

採点・面接のたびに、Googleスプレッドシートに **1行ずつ履歴が溜まる** ようにします。
データベースは不要。奥様と共有すれば、二人ともいつでも全履歴を見られます。

所要時間：10〜15分くらい。

---

## 全体の流れ

```
①スプレッドシートを1枚作る
   ↓
②[拡張機能]→[Apps Script] にスクリプトを貼る
   ↓
③「ウェブアプリ」としてデプロイ → URL を発行
   ↓
④その URL を Render の環境変数 SHEET_WEBHOOK_URL に入れる
   ↓
⑤シートを奥様に共有する
```

---

## ① スプレッドシートを作る

- ブラウザで **https://sheets.new** を開く（新しい空のシートができます）
- 名前を「**編入チャレンジ 履歴**」などに変更（任意）
- ※ 奥様がメインで見るなら、**奥様のGoogleアカウントで作る**と管理がラク。
  あなたが作って後で共有してもOK（⑤で共有します）。

## ② スクリプトを貼り付ける

1. シートのメニュー **[拡張機能] → [Apps Script]** を開く
2. 最初から入っている `function myFunction() {}` を**全部消す**
3. このフォルダの **`スプレッドシート用スクリプト.gs`** の中身を**全部コピーして貼り付け**
4. 上の **保存アイコン（💾）** を押す

## ③ ウェブアプリとしてデプロイ（URLを発行）

1. 右上の **[デプロイ] → [新しいデプロイ]**
2. 左の歯車⚙ → **[ウェブアプリ]** を選ぶ
3. 設定：
   - **説明**：なんでもOK（例「henyu log」）
   - **次のユーザーとして実行**：**自分**
   - **アクセスできるユーザー**：**全員**（← これ重要。Renderから書き込めるように）
4. **[デプロイ]** を押す
5. **「承認が必要」** と出たら：
   - **[アクセスを承認]** → 自分のGoogleアカウントを選ぶ
   - 「このアプリはGoogleで確認されていません」と警告が出ます（自作なので正常）
     → **[詳細]** → **[（プロジェクト名）に移動（安全ではないページ）]** → **[許可]**
6. **ウェブアプリのURL**（`https://script.google.com/macros/s/……/exec` で終わる）が表示される
   → **コピー**しておく

> ⚠️ URLは末尾が **`/exec`** のものを使います（`/dev` ではない方）。

## ④ Render に URL を登録

1. https://dashboard.render.com/ → **Senrikokusai** → 左メニュー **[Environment]**
2. **[Add Environment Variable]**：
   - **Key**：`SHEET_WEBHOOK_URL`
   - **Value**：③でコピーしたURL（`…/exec`）
3. **[Save Changes]** → 自動で再デプロイ（数分）

## ⑤ シートを奥様に共有

- スプレッドシート右上の **[共有]** ボタン
- 奥様（や自分）のメールアドレスを追加 → **[閲覧者]** か **[編集者]** → 送信
- これで二人ともスマホ・PCから履歴を見られます

---

## 記録される内容（列）

| 日時 | 種別 | 言語 | お題・質問 | 回答 | 点数 | 評価 | コメント | 詳細 |
|---|---|---|---|---|---|---|---|---|

- **種別**：小論文 / 面接
- **詳細**：小論文は各観点の点数・良い点・改善点、面接は「○問目」

---

## うまく動かないとき

- スクショや状況を伝えてください。こちらから接続テスト（`/api/_sheettest`）を実行して原因を確認します。
- よくある原因：
  - ③で「アクセスできるユーザー＝**全員**」になっていない
  - URLが `/exec` ではなく `/dev` になっている
  - Renderで `SHEET_WEBHOOK_URL` の保存後、再デプロイが終わる前にテストした（数分待つ）
